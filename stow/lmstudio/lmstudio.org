#+TITLE: LM Studio for local AI

Learning to use LM Studio for local AI management, see [[https://lmstudio.ai]].

That runs as a local API server that loads LLMs when they are requested.



* Systemd user service

Starts as a systemd user service on login, loading models on demand.

#+begin_src systemd :tangledir :tangle .config/systemd/user/lmstudio.service :mkdirp yes :padline no
[Unit]
Description=LM Studio (selfhosted LLM) server
Documentation=https://lmstudio.ai/
After=network.target

[Service]
Type=forking
ExecStart=/home/jiby/.lmstudio/bin/lms server start
ExecStop=/home/jiby/.lmstudio/bin/lms server stop
Restart=on-failure
# Don't restart if exited OK
RestartPreventExitStatus=0

[Install]
WantedBy=default.target
#+end_src

Get NPM off homebrew:

#+begin_src shell
brew install npm
#+end_src


#+begin_src shell
npm install -g @mariozechner/pi-coding-agent
#+end_src

Perfect provider

#+begin_src shell
pi install git:github.com/vampaz/pi-custom-provider-lmstudio@master
#+end_src

Use envvar for actual endpoint URL:
#+begin_src shell
export LMSTUDIO_ENDPOINT_URL=http://localhost:8001
#+end_src

And now that works, install an ACP client: pi-acp

#+begin_src shell
npm install -g pi-acp
#+end_src
